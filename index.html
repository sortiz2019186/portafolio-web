Aplicación "Tony's Kinal"

Esta es una aplicación de escritorio hecha con MySQL, para la base de datos, con Java Standard Edition y JavaFX Scene Builder para el diseño de cada una de las vistas de la aplicación.
Esta aplicación fue pensada y hecha para poder llevar el control en un "restaurante". Al iniciar la aplicación se puede notar que existe una pequeña barra de navegación con los apartados "Módulos", "Reportes", "Entidades de unión" y "Acerca de".

Módulos

Al dar clic en este apartado, se desplagará un menú con diferentes opciones para seleccionar: Se muestran nombres de entidades las cuales no tienen algun relación o depencia de algun dato de otra entidad.
Al seleccionar cualquiera de las vista de entidades, le mostrará una nueva ventana de la siguiente forma: Este nueva ventana, contiene una cuadros de texto para ingresar la información, una tabla en la que se muestran los datos existentes en la base de datos y botones con los cuales podrá crear, eliminar, editar y generar reportes.

Crear

Al dar clic en el botón nuevo, podrá podrá crear un nuevo dato. Deberá ingresar la información correspondiente en cada uno de los cuadros de texto. Cabe resaltar un par de cosas: 1. El ID se genera automáticamente en la base de datos, por esa razón el cuadro de texto no es habilitado; 2. Los bonotes de "nuevo" y "eliminar" cambiar dinámicamente par apoder "guardar" o "cancelar" la acción de crear un dato: Al dar clic en guardar, el dato se actualizará de manera inmediata tanto en la tabla como en la base de datos de la aplicación. Al cancelar la acción, no se actualizará nada dentro de la aplicación. Al seleccionar cualquiera de las dos acciones, los botones volveran a su estado original.

Eliminar

Para poder realizar esta acción, se debe seleccionar el dato a eliminar, de lo contrario, saldrá el siguiente mensaje: Al tener ya un dato seleccionado y dar el en bontón de eliminar, le mostrará un mensaje de confirmación para poder eliminar el dato seleccionado: Como se puede apreciar, al seleccionar un dato este mismo es reflejado en los cuadros de texto y en su campo correspondiente. Si el usuario está seguro de eliminar el dato, al presionar sí, el dato será removido tanto de la tabla y de la base de datos de manera inmediata. Caso contrario, al seleccionar no, ningún cambio será efectuado en la apicación.

Editar

De la misma manera que en la acción de "eliminar", antes de dar en el botón de eliminar se debe seleccionar un dato. Al tener el dato seleccionado y dar en el botón editar, los cuadros de texto se habilitarán para poder editar el dato deseado en los cuador de texto correspondientes: Una vez los datos modificados, usted podrá dar en el botón de actualizar para que los cambios sean aplicados y sean reflejados tanto en la aplicación como en la base de datos. Al seleccionar cancelar ningún cambio será apicado en la apicación.

Aplicación web
"Sucursales McDonalds"

Esta es una aplicación web realizada con tecnologías como Angular, NodeJS, Express y MongoDB, fosteada en Firebase y el código backend subido y corriendo en Heroku. Pensada en llevar un mejor control con todas las sucursales de la empresa McDonalds, por ejemplo, los productos que la empresa en general posee, o bien, los productos que se envían y tiene cada sucursal de la empresa.

La aplicación a primera instancia, cuenta con una barra de navegación en la que se puede ver el logo de la empresa a la que pertenece la aplicación. También, cuenta con un botón para iniciar sesión, el cual redirige al usuario a una nueva pestaña en la que puede ingresar sus credenciales y autenticarse. Al lado, puede verse un botón de "acerca de" el cual despliega una ventana emergente en la que se muestran a detalles los datos de la persona quien desarrolló la aplicación.

Barra de navegación - bonotes

Pestaña "iniciar sesión"

Ventana emergente "Acerca de"

A la hora que el usuario administrador ingrese a la aplicación, podrá tener el manejo de todos lo datos y diferentes tablas que contenga la misma, por lo tanto, puede crear, eliminar, modificar y ver a detalle los datos que contenga la tabla. De igual forma, en la tabla se muestran todos los datos en general y las acciones que en administrador puede realizar con cada uno de los datos.
La barra de navegación dinámica, permite validar que cuando exista un usuario autenticado, ya no le muestre los bonotes de "iniciar sesión" ó "Acerda de", sino, le muestra bonotes con los cuales puede dirigirse a las tablas existentes y un menú desplegable para poder administrar su perfil o cerrar la sesión en la aplicación.

Descripción de las "acciones""

Crear

El administrador podrá crear o agregar datos en la base de datos al presionar el botón "crear sucursal". Este mismo le desplegará un formulario en el cual el administrador ingresará todos los datos que se le solicitan. Ejemplo: Si el usuario opta por guardar los datos ingresados, estos mismo se verán reflejados en la tabla y al mismo tiempo en la base de datos. Si el usuario decide no guardar los datos, presiona cancelar y no sucede algún cambio.

Editar

El usuario administrador podrá editar los datos que él desee, esto por medio de los bonotones incluidos en los items del dato. Al momento de editar, se le desplegará una ventana emergente con los datos actuales, los cuales podrá cambiar: Hay datos que no se pueden editar, esto debido a que estos datos hacen posible la relación entre algunas tablas.
Si el usuario decide guardar los cambios estos mismos se verán reflejados tanto en la tabla como en la base de datos de manera inmediata. Si decide no conservarlos, bastará con presionar el botón cerrar y no sucederá cambio alguno.

Eliminar

El administrador podrá eliminar el dato que desee, esto también es posible por medio de los botones incluidos en los items del dato. Al momento de eliminar el dato seleccionado, se le mostrará a detalle todos los datos correspondientes al que seleccionó: Si el usuario decide eliminar el dato, se le desplegará una ventana emergente de confirmación para la eliminación del dato, mostrando el nombre de dato a eliminar: Si presiona eliminar se eliminará el dato tanto de la tabla con en la base de de datos. Si decide lo contrario, bastará con presionar cancelar y no se aplicará ningún cambio a la tabla y base de datos.
